What this table does
--------------------
T4 is the IDIOM and COLLOCATION table.  After T1-T3 have resolved the
grammatical structure, T4 handles:

  1. Fixed English expressions → Russian equivalents (backtick literals)
     e.g. `need` → xпонадобится, `be sure` → Vубедитесь, `both...and` → как...так и
  2. Final Z cleanup in positions not handled by T1-T3
  3. Negation normalisation: `no` → K, K + comma/boundary → нет
  4. Structural markers: b (imperfective 'to') + context → case prepositions
  5. Parenthesis / bracket handling: (#) → flattened form
  6. Guard rules (none-action) for structures already assembled by T1-T3
     that must not be re-processed (30+ no-action rules)
  7. Adjectival compounds: A-A-N, A-N, H-N (hyphenated noun phrases)
  8. Special constructions: "there is/are", "it is", "let us", percentages

Record size is 9 bytes (not 10 like T1-T3) — flags field is 1 byte at offset 0,
not 2 bytes at offset 8. The compiler likely zero-extends this to 16 bits
(0x3C in T4 = 0x003C in T7/T8). Dual role: priority/ordering + constituent-type
index for the compiler (same index space as other tables).
This is the most structurally heterogeneous table.

Pattern syntax clues
--------------------
  `word`    = exact English word match (backtick-quoted)
  !word!    = inline Russian literal injection (e.g. !мес! = месяц)
  x         = impersonal/existential verb combination
  p         = resolved preposition (lowercase P, already processed)
  W         = multi-word phrase marker (from T3)
  *         = sentence boundary OR wildcard (context-dependent)
  %         = modifier flag (e.g. <H%D,> = digits with modifier)

Concrete examples
-----------------

**Rule [1]** `X`need`` → `` `xпонадобится` ``

Pattern: `X` (auxiliary 'be') `` `need` `` (literal word)

Example: *"is needed"*
Token stream: `[Xis] [Eneeded]`
But the pattern needs the literal word "need", not its past-participle "needed".
Better example: *"is need"* (archaic/formal):
Token stream: `[Xis] [Nneed]`
Match at position 1:
  `X`       → ts[1]="Xis..." starts with X → match → rep=`` `xпонадобится` `` → replace token 1
  `` `need` `` → ts[2]==en_ru["need"] → match → rep=(none, but lit replacement already applied)
Result: the X auxiliary and "need" are replaced by the Russian idiom "понадобится"
(roughly "will be needed"). This handles constructions like "is need of" → "нуждаться в".

---

**Rule [26]** `N<"'#>N` → `A`

Pattern: `N` (noun) `<"'#>` (optional quotes, apostrophes, numbers) `N` (noun)

Example: *"field west"*
Token stream: `[Nfield] [Nwest]`
Match at position 1:
  `N`     → ts[1]="Nfield..." starts with N → match → rep=`A` → replace with A (adjective)
  `<"'#>` → no tokens between → zero-wide
  `N`     → ts[2]="Nwest..." starts with N → match → rep=(none, but already consumed)
Result: "field" becomes A (adjective). In "field west of a house", "field" modifies
"west" attributively → Russian reads "полевой запад" (field-west as adjective-noun).

---

**Rule [69]** `K`as`A`as`` → `` `Dне так`A`Jкак` ``

Pattern: `K` (not) `` `as` `` (literal) `A` (adjective) `` `as` `` (literal)

Example: *"not as big as"*
Token stream: `[Knot] [Das] [Abig] [Das]`
Match at position 1:
  `K`       → ts[1]="Knot..." starts with K → match → rep=`` `Dне так` `` → "не так"
  `` `as` ``  → ts[2]==en_ru["as"] → match → rep=`A` → keep adjective marker
  `A`       → ts[3]="Abig..." starts with A → match → rep=`` `Jкак` `` → "как"
  `` `as` ``  → ts[4]==en_ru["as"] → match → rep=(consumed)
Result: "not as big as" → "не так big как" in Russian output (not so big as).

---

**Rule [85]** `` `no`[,C*] `` → `K`

Pattern: `` `no` `` (literal "no") `[,C*]` (comma, conjunction, or sentence-end)

Example: *"no,"*
Token stream: `[kno] [, ]`
Match at position 1:
  `` `no` `` → ts[1]==en_ru["no"] → match → rep=`K` → replace with negation tag K
  `[,C*]`  → ts[2]=", " starts with `,` → match
Result: "no" becomes K (negation particle). Later rules (like T4[86]) will
turn K at clause-end into the Russian word "нет".

---

**Rule [86]** `K[,C*]` → `` `нет` ``

Pattern: `K` (negation) `[,C*]` (comma/conjunction/boundary)

Example: *"No."*
Token stream: `[Kno] [..]`
  (After T4[85] has already converted "no" → K)
Match at position 1:
  `K`     → ts[1] starts with K → match → rep=`` `нет` `` → "нет"
  `[,C*]` → ts[2]="." is `*` boundary → match
Result: "No." → "нет." The standalone negation particle outputs the Russian word "нет".
